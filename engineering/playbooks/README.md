# Playbooks

Step-by-step operational sequences that combine prompts/scenarios with actual
commands — things you run before an event (a release) rather than reference
material you read (that's what `docs/` is for).

## Index

- `release-checklist.md` — what to run through before promoting a build to
  `www.thefarmflow.in` using the staged-deploy flow in
  [CLAUDE.md](../../CLAUDE.md#release-process).

This is deliberately thin at the moment — add a playbook here when a
recurring "before I do X" sequence starts living only in your head.
